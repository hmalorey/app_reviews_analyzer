### --- REPOSITORIES --- ###
A repository is a container for a project that is tracked by backed.
You have local repositories on your computer.
You have remote repositories, like Github.com


#### --- KEY COMMANDS --- ####

### Connaitre la version de Github
git --version

### Faire d'un dossier existant un repository (track changes via git)
(cela créé differents fichiers cachés qui vont tracker les chgts, dont un .git folder)
git init

## Savoir si des chgts ont été faits sur le repo mais n'ont pas été commit sur la branche
git status

#### ----- HOW TO COMMIT ? TWO STEP PROCESS ---- ####
1/ Add to staging file
git add filename1.py filename2.py 
or to add all:
git add .

2/ Commit
git commit -m "Message string"

## Récupérer l'historique des commits (avec leur id pour faire des gco)
git log

### Go back to a past version of our code on master/develop local branch
git check out <commit_id>

## Changer de branche
git checkout <branchname>

## --- BRANCHES ----- ##
Branch that is used when others view your code on Github.com
Branch to which the commits are done
Documentation : https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/about-branches

git branch <branch_name>

### Liste des branches en local ##
git branch

### Liste des branches en remote ##
git branch -r

### Liste des toutes les branches
git branch -a

## Merger du code sur master (se mettre sur master)##
git merge <branchname>

=> tous ces changements sont fait en local, en utilisant GIT -- ET NON GITHUB.

#### ++++ GITHUB TERMINOLOGY +++ #####

upstream <= The remote git repo (likely whose master or release branch is in production)

forked-repo <= The remote [experimental git repo] (https://docs.github.com/en/github/getting-started-with-github/fork-a-repo) also known as "origin".

local repo <= The files and directories that you work with on your local workstaion, which you likely got by running a git clone my-forked-repo.git command

local index <= Also known as your local git "stage", i.e., where you stage your files before pushing them to you remote repo.

### -- Comment UPLOADER des fichiers de code pour la première fois sur Github ? --- ####
1/ Créer le repo sur Github
2/ En local, aller dans dans le dossier à uploader et faire git init
3/ En local, ajouter les files sur staging local en faisant git add .
Option: être sur qu'on est sur master en faisant : git branch -M main
4/ En local, commit les files sur la branche locale main en faisant git commit -m "commit message"
5/ Ensuite, paramétrer l'URL remote (appelée origin) a laquelle on veut pousser le code ensuite en faisant: 
git remote add origin https://github.com/hmalorey/a_key_commands.git
6/ Pousser le code sur Github main en faisant : 
git push -u origin main


### -- Comment mettre à jour le code d'un repo deja existant sur Github ? --- ####
1/ En local, ajouter les files sur staging local en faisant git add .
2/ En local, commit les files sur la branche locale main en faisant git commit -m "commit message"
3/ Checker quelle adresse remote est paramétrer en faisant
git remote -v
Si ce n'est pas la bonne (celle du repo cible sur Github), la mettre a jour en faisant:
git remote set-url origin https://github.com/hmalorey/backtest/trix.git
4/ Pousser le code sur main en faisant: -
git push -u -f origin main
